1. Introdução

O Asciidoctor é uma ferramenta bastante eficiente para a elaboração de sites [1] sem a necessidade de conhecer linguagens para ambiente web como HTML e CSS. Sua funcionalidade é bastante extendida, englobando desde a formatação de textos e tabelas até a inserção de mídias de áudio e vídeo. Um amplo tutorial é fornecido no site oficial. No entanto, por abranger tantas informações, às vezes torna-se necessário um material mais conciso, com as funções mais utilizadas. Esse modelo tem o objetivo de fornecer isso, incluindo os elementos mais comuns no dia a dia de criação de sites com o Asciidoctor.

Dessa forma, caso uma dúvida simples aconteça, basta verificar esse modelo. Contudo, certamente haverá casos mais complexos que serão solucionados de maneira mais eficiente através da leitura do tutorial oferecido pela equipe do Asciidoctor. Assim como na linguagem Latex, o objetivo do Asciidoctor é trabalhar com conteúdo sem se preocupar com a apresentação, pois a mesma já está pronta para uso. Então, mãos à obra!

2. Formatação de Texto

2.1. Atributos de Formatação

Imagine que você deseja destacar alguma palavra no seu texto. O Asciidoctor fornece algumas opções para isso, como por exemplo: mudar o tipo de fonte, tamanho e cores. Veja abaixo um exemplo autoexplicativo dessas opções:

Código
[underline red]#Vermelho Sublinhado#									+ (1)
[overline yellow-background small]#Letras Pequenas com Fundo Amarelo#	+
[line-through green big]#Letras Grandes Verdes#							+
Resultado

Vermelho Sublinhado
Letras Pequenas com Fundo Amarelo
Letras Grandes Verdes

1 O sinal + indica uma quebra de linha.

As cores suportadas são definidas pelo padrão HTML. Veja mais aqui.

Para diferentes fontes, como itálico e negrito, o Asciidoctor define a seguinte sintaxe:

Código
_Texto em Itálico_.                   +
*Texto em Negrito*.                   +
__A__penas primeira letra em itálico. +
**A**penas primeira letra em negrito. +
Resultado

Texto em Itálico.
Texto em Negrito.
Apenas primeira letra em itálico.
Apenas primeira letra em negrito.

2.2. Sobrescritos e Subescritos

Quando desejamos inserir Sobrescritos e Subescritos, sem usar os comandos latex, podemos escrever da seguinte forma:

Código
1000 = 10^3^.                     + (1)
e^&#960;i^.                       + (2)
a^_x_^ * a^_y_^ = a^(_x_+_y_)^.   +
Prof^a^ Maria.                    +
                                  +
H~2~O.                            + (3)
X~k~ = X~k~ + X~k-1~.
Resultado

1000 = 103.
eπi.
ax * ay = a(x+y).
Profa Maria.

H2O.
Xk = Xk + Xk-1.

1 O conteúdo que deve estar no expoente é inserido entre ^ ^.
2 É possível o uso de caracteres especiais do HTML .
3 A inserção de subescrito é realizada da mesma forma, apenas trocando ^ ^ por ~ ~.

3. Formatação de Parágrafos

3.1. Parágrafo Normal

Se você já produziu algum texto em Latex, irá reconhecer imediatamente a estratégia do Asciidoctor. Assim como no Latex, para iniciar um novo parágrafo, basta introduzir uma linha em branco antes de iniciar o texto. Simples .

3.2. Versos

Você está em um dia inspirado, contemplando a natureza e pensando sobre a beleza da vida. Então, decide fazer um poema e divulgar no seu site feito em Asciidoctor. Para isso, basta escrever seu poema da seguinte forma:

Código
[verse, William Blake, from Auguries of Innocence]
To see a world in a grain of sand,
And a heaven in a wild flower,
Hold infinity in the palm of your hand,
And eternity in an hour.
Resultado
To see a world in a grain of sand,
And a heaven in a wild flower,
Hold infinity in the palm of your hand,
And eternity in an hour.
— William Blake
from Auguries of Innocence

3.3. Citações

Para citar uma frase célebre, e destacá-la de um modo elegante, pode-se fazer:

Código
[quote, Bertrand Russel, The World of Mathematics (1956)]
A good notation has subtlety and suggestiveness wich at times makes it almost seem like a live teacher.
Resultado
A good notation has subtlety and suggestiveness wich at times makes it almost seem like a live teacher.
— Bertrand Russel
The World of Mathematics (1956)

3.4. Advertências

Advertências são uma boa forma de chamar atenção do leitor para algum detalhe específico. Podemos inserir advertências incluindo algumas palavras no início do parágrafo. As opções são:

Código
TIP: Dica.

NOTE: Nota.

WARNING: Importante.

CAUTION: Cuidado.
Resultado

Dica.
Nota.
Importante.
Cuidado.

4.5. Notas de Rodapé

Há duas formas de inserir uma nota de rodapé. A primeira forma é usada quando precisamos citar a nota apenas uma vez:

Código
Eis aqui um exemplo de nota de rodapé footnote:[Um exemplo muito bem feito, por sinal.].
Resultado

Eis aqui um exemplo de nota de rodapé [2].

Quando imaginamos que a nota será citada mais de uma vez ao longo do texto, a sintaxe muda um pouco. Veja como fica:

Código
Mais uma vez, preciso inserir uma nota de rodapé footnoteref:[note3, E o exemplo só melhora...].
Como prometido, posso citar novamente a nota de rodapé footnoteref:[note3].
Resultado

Mais uma vez, preciso inserir uma nota de rodapé [3]. Como visto antes, posso citar novamente a nota de rodapé [3].

4.6. Notas Explicativas

Notas Explicativas [4] são muito úteis para demonstrar ao leitor detalhes de algum código, etc. Veja o exemplo a seguir:

Código
                                                                                    +
[source,c]                                                                          +
----                                                                                +
#include <stdio.h>                                                                  +
                                                                                    +
int main(){                     // <1>                                              +
    printf("Hello World! \n");  // <2> <3>                                          +
    return 0;                   // <4>                                              +
}                                                                                   +
----                                                                                +
                                                                                    +
<1> Definição do início do programa.                                                +
<2> A função printf imprime uma mensagem no terminal.                               +
<3> O caractere \n indica o fim da linha.                                           +
<4> Se o programa for executado corretamente, sinaliza o fim retornando o valor 0.  +
Resultado
#include <stdio.h>

int main(){                     (1)
    printf("Hello World! \n");  (2) (3)
    return 0;                   (4)
}
1 Definição do início do programa.
2 A função printf imprime uma mensagem no terminal.
3 O caractere \n indica o fim da linha.
4 Se o programa for executado corretamente, sinaliza o fim retornando o valor 0.

Para ter notas explicativas dentro de códigos incluídos como arquivos, basta lembrar de adicionar os <n> identificadores dentro de comentários no código.

5. Links

Quantas vezes nós precisamos inserir um link que faça referência a um site externo ou a outro tópico do nosso próprio texto não é mesmo? Tendo isso em vista, o Asciidoctor dispõe de maneiras bastante práticas para o trabalho com links e referências internas. Veremos aqui as formas mais simples de criar links, mas caso você deseje mais informações a respeito, basta visitar o manual do Asciidoctor.

5.1. Sites

O Asciidoctor possui suporte para a criação automática de links com domínio http, ftp, irc e emails. Assim, basta adicionar o link no seu texto que ele será criado. No entanto, muitas vezes não queremos que o link seja o próprio nome do site, e sim que uma palavra ou figura seja o objeto clicável que encaminhará o usuário para um suposto site. A forma de realizar este tipo de modificação pode ser vista abaixo:

Código
:linkattrs: (1)

Para visitar a homepage do Asciidoctor, clique http://asciidoctor.org/[aqui].              +
Para visitar a homepage do Asciidoctor em nova aba, clique http://asciidoctor.org/[aqui^]. +
Para acessar o facebook: https://facebook.com/[icon:facebook[]]
Resultado

Para visitar a homepage do Asciidoctor, clique aqui.
Para visitar a homepage do Asciidoctor em nova aba, clique aqui.
Para acessar o facebook:

1 Devemos inserir esse atributo no início do documento para habilitar algumas funcionalidades como, por exemplo, abrir o link em nova aba.

5.2. Referências Internas

Referências internas interligam diferentes pontos dentro do texto. No caso de títulos de seções e subseções, os próprios títulos servem para fazer a referência. Já no caso de códigos, figuras, tabelas, fórmulas, etc., é necessário a criação de de uma referência. O procedimento a ser realizado é basicamente o seguinte:

Código
Link para <<Links>>.


[[codigo_exemplo]] (1)
.Código            (2)
[source,c]         (3)
----                                +
#include <stdio.h>                  +
                                    +
int main(){                         +
    printf("Hello World! \n");      +
    return 0;                       +
}                                   +
----                                +

Link para o <<codigo_exemplo,código>>. Outra forma: xref:codigo_exemplo[código] (4)
Resultado

Link para Links.

Listagem 1. Código
----                                +
#include <stdio.h>                  +
                                    +
int main(){                         +
    printf("Hello World! \n");      +
    return 0;                       +
}                                   +
----                                +

Link para o código. Outra forma: código

1 Definição do nome que será usado para a referência
2 Legenda do bloco.
3 Definição do bloco. Nesse caso é um bloco de código, mas também pode ser de imagem, vídeo, etc.
4 Observe como os links podem ser criados.

Como sempre, mais informações podem ser encontradas no manual do Asciidoctor.

6. Códigos

A inserção de códigos em um site pode ser realizado de maneira simplista, tratando-se apenas da configuração de um bloco. O código pode ser inserido manualmente, ou através da diretiva include. Veja o exemplo abaixo:

Código
:source-highlighter: pygments  (1)

[[codigo]]                     (2)
[source,cpp,options="nowrap"]  (3)
.Exemplo de código.
----                           +
include::./codigos/main.cpp[]  +
----                           +
Resultado
Listagem 2. Exemplo de código.
#include <opencv2/opencv.hpp>
#include <iostream>

using namespace cv;
using namespace std;

int main( int argc, char** argv ){
    Mat image;
    image = imread(argv[1], CV_LOAD_IMAGE_COLOR);

    namedWindow( "Display window", WINDOW_KEEPRATIO );
    imshow( "Display window", image );

    waitKey(0);
    return 0;
}
1 Inserindo esse atributo no início do arquivo .adoc, os códigos terão suas palavras chaves com cores diferentes.
2 Definição da palavra para referência interna.
3 A inclusão do atributo options="nowrap" permite que uma barra de rolagem seja criada no final do bloco de código para os casos em que uma linha de código é maior que a largura da tela do computador.

7. Imagens

Imagens, assim como os código vistos na seção anterior, podem ser incluídas através de um simples bloco. Mas outras opçẽs de customização estão disponíveis, como veremos a seguir.

7.1. Estrutura Padrão

Código
:imagesdir: ./imagens                (1)

[[eiffel]]                           (2)
[.text-center]						 (3)
.Torre Eiffel                        (4)
[link=https://www.google.com/eiffel] (5)
image::eiffel.jpg[]
Resultado
Figura 1. Torre Eiffel
1 A inserção desse atributo facilita a inserção de imagens, pois assim não é necessário oferecer o caminho completo na diretiva include.
2 Definição de um nome para uso em referências internas.
3 Atributo para definir a posição da imagem no conteúdo.
4 Título da imagem, que será usado como legenda.
5 Caso essa linha seja adicionada, a imagem se comportará como um link quando clicada.

7.2. Controlando Tamanho e Posição

No exemplo anterior, vimos que a imagem ocupa um espaço considerável. E se você quiser definir o tamanho exato da imagem? O Asciidoctor também oferece a possibilida de definir o tamanho de imagens. Veja:

Código
[[eiffel2]]
[.text-center]
.Torre Eiffel
image::eiffel.jpg[450,450] (1)


[[eiffel3]]
[.text-center]
.Torre Eiffel
image::eiffel.jpg[40%,40%] (2)
Resultado

Figura 3. Torre Eiffel

2 Definindo o tamanho em relação ao tamanho da página, em %.
